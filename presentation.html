Definition

What is an EVO?

An Evolving Value Object is a Program Derived Account on Solana that:

1️⃣
Holds locked SOL — the lamports literally live inside the PDA's account balance
2️⃣
Generates evolving art — computed client-side from on-chain data that changes over time
3️⃣
Has a price floor — shatter it to reclaim the locked SOL. Floor = locked value.
4️⃣
Is trustless — no admin, no update authority, no metadata. Art is pure math.
5️⃣
Records history — every trade leaves a permanent mark (fracture line) in the art

Mechanics — Part 1

Forge & Feed

🔨 Forging (Minting)

Pay a small fixed price to create an empty z PDA.

forge(collection_id)

// User pays: 0.05 SOL
// → 0.01 SOL: rent (stays in PDA)
// → 0.04 SOL: collection treasury

// Result: empty z
// locked_lamports = 0
// facet_count = 0
// resonance_seed = hash(tx_sig)

💎 Feeding (Depositing)

Deposit SOL into your z. This is the store of value action.

feed(z_id, amount)

// User feeds: 1 SOL
// → 1 SOL added to locked_lamports
// → z grows visually
// → Fee: 0 SOL (frictionless)

// No minimum. No maximum.
// No lockup. Feed anytime.

Mint cheap, feed over time. A z fed 0.1 SOL weekly for a year tells a different story than one fed 50 SOL in one shot — even if the total is the same.

Mechanics — Part 2

Grow & Trade

⏳ Growing (Time Evolution)

Facets increase automatically over time. No transaction needed.

// Computed from forged_at:
facet_count = floor(
  (now - forged_at) / 7_days
)

// 1 week old → 1 facet
// 1 year old → 52 facets
// Capped at 100 facets

// Trading doesn't reset
// the timer. Age is total.

🔄 Trading (Marketplace)

Seller lists, buyer pays, ownership transfers. Built into the program.

list(z_id, price)
buy(z_id)

// Buyer pays X SOL
// → Seller: X - 2%
// → Protocol: 1% of X
// → Collection: 1% of X
// → Fracture line recorded
// → trade_count++

Every trade leaves a fracture line — a permanent visual mark. A z traded 20 times has 20 fracture lines. A z never traded is pristine. Both are beautiful in different ways.

Mechanics — Part 3

Shatter (The Exit)

Destroy the z. Reclaim your SOL. Art gone forever.

shatter(z_id)

// Program returns ALL locked_lamports to owner
// → 99% to owner
// → 1% to protocol treasury
// → PDA closed (account destroyed)
// → Art, history, facets — all gone
// → z ceases to exist
📉

Price Floor

If market price drops below locked SOL, someone arbitrages by shattering. Floor = locked SOL.

📉

Decreasing Supply

Shattering reduces total supply. Over time, Z are destroyed, making survivors more rare.

♾️

Irreversible

Once shattered, the resonance seed is gone forever. A new z gets a new seed. No undo.

Technical — Trustless

Zero Trust Required

What doesn't exist

  • ✗ No admin key
  • ✗ No update authority
  • ✗ No metadata URI
  • ✗ No mint authority (after cap)
  • ✗ No freeze authority
  • ✗ No upgrade authority (post-deploy)
  • ✗ No pausing mechanism

What protects the SOL

  • ✓ Program logic (only owner can shatter)
  • ✓ Program logic (only owner can feed)
  • ✓ Program logic (only owner can list/delist)
  • ✓ Program logic (buyer pays → ownership transfers)
  • ✓ PDA has no private key (can't be stolen)
  • ✓ Upgrade authority revoked (code is immutable)
  • ✓ Art is deterministic (can't be changed manually)

The SOL is protected by math and program logic, not by trust in any person or entity. That's the Solana way.
